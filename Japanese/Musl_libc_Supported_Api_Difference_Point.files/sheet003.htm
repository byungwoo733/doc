2. スレッドAPI の Manpage に定義テイギされる動作ドウサは以下イカの通トオりです。
アプリケーションが pthread_attr_setstack() によってスレッドスタックを割ワり当アてるとき、guard size 属性ゾクセイは無視ムシされます（libcによってガード領域リョウイキは作成サクセイされません）。スタックのオーバーフローは（mprotect(2) でスタック用に確保した領域の終端にガード領域を指定するなどして）アプリケーションの責任セキニンで対処タイショする必要ヒツヨウがあります。
VE では以下イカの様ヨウに動作ドウサが異コトなります。
ガード領域リョウイキが（ユーザによって）指定シテイされた後アトであっても、スタックは領域リョウイキの終端シュウタンを超コえて伸長シンチョウし続ツヅけます。その結果ケッカは不定フテイです。そのため、pthread_attr_setstack() は使ツカうべきではありません。

アプリケーションが子コスレッドのスタックサイズを pthread_attr_setstacksize(3) で指定シテイした場合バアイは、この問題モンダイは発生ハッセイしません。
